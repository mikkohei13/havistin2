`/havis` is a login-protected Flask page served by [`app/routes/havis.py`](app/routes/havis.py) (`havis_root`) and rendered with [`app/templates/havis.html`](app/templates/havis.html).

The feature is in a demo phase. No backwards compatibility is needed. Keep it simple.

Frontend logic in [`app/static/app/havis.js`](app/static/app/havis.js) handles the flow: **Aloita** starts a **havaintoerΟΛ** (IndexedDB `sessions` row with `startedAt`), then `geolocation.watchPosition`. **Lopeta havaintoerΟΛ** sets `endedAt`. **LisΟΛΟΛ havainto** starts MediaRecorder, recording auto-stops at 20s, then audio + lat/lng are posted as multipart to `/havis/api/transcribe`; saved observations include `sessionId` / `sessionStartedAt`.

Backend **`havis_transcribe`** in [`app/routes/havis.py`](app/routes/havis.py) validates auth/input, transcribes Finnish speech via OpenAI Audio API (`client.audio.transcriptions.create`, model `gpt-4o-transcribe`), then matches the first transcribed word against a small fixed species-name list in [`app/routes/havis_species_names.py`](app/routes/havis_species_names.py) (Levenshtein distance, tie βÜ£ list order; skip hint if edit ratio would be βâΞ 30%) and passes transcript + optional hint in the **user** message to GPT. Structured extraction lives in [`app/havis_structurization.py`](app/havis_structurization.py) (`structurize_transcript`, `GPT_MODEL`), which returns JSON fields such as species, count, notes, `raw_transcript`, warnings.

Persistence is browser-side IndexedDB in `havis.js` (`havis_db`, store versions use `DB_VERSION` in `havis.js`; stores `sessions` and `observations`); no server DB. Each `sessions` row includes `userId` (from `session["user_data"]["id"]`, injected as `window.HAVIS_USER_ID`) so an open erΟΛ is scoped to the logged-in user. On load, any open session for that user (`endedAt` unset) is resumed (GPS + UI); `localStorage` key `havis_active_session` mirrors the active session for the same user and is cleared when another accountβÄôs pointer is detected or when **Lopeta havaintoerΟΛ** confirms. Only **Lopeta havaintoerΟΛ** + confirm ends an erΟΛ (`endedAt`). Observations are listed from IndexedDB; row click opens a modal with details + Leaflet map marker.

### FinBIF Vihko submission

For a **closed** havaintoerΟΛ (`endedAt` set), each session group in the list shows **LΟΛhetΟΛ Vihkoon** (the list refreshes as soon as the erΟΛ is ended so the button appears without reloading the page). The client POSTs JSON to **`POST /havis/api/submit`** with `{ session, observations }` from IndexedDB (same-origin cookie session). After a **successful** response, that session and all its observations are **removed** from IndexedDB and disappear from the UI.

Server-side submission is implemented in [`app/havis_finbif_submit.py`](app/havis_finbif_submit.py): builds a FinBIF **store-document** (fixed **collection** `HR.1747`, **form** `JX.519` βÄî Vihko notebook), resolves Finnish species names to taxon IDs via `GET /autocomplete/taxa`, then calls **`POST /documents/validate`** and only if that succeeds **`POST /documents`**. Uses [`app/helpers/common_helpers.py`](app/helpers/common_helpers.py) for FinBIF HTTP (Bearer `FINBIF_API_TOKEN` from env via [`app/app_secrets.py`](app/app_secrets.py), `Person-Token` from the logged-in user). The logged-in personβÄôs **`id`** from `user_data` is used in `gatheringEvent.leg`. Backend logs include progress and the document sent to validate (see `Havis FinBIF info:` / `Havis API /api/submit` lines on stdout).

Styles are in [`app/static/app/havis.css`](app/static/app/havis.css). Status text is split: `#gps_status` for geolocation messages, `#session_status` for an ongoing havaintoerΟΛ (start time + βÄ€minuuttia sittenβÄù, updated periodically), and `#ui_status` for recording/transcription/IndexedDB, FinBIF submit, and other process UI. OpenAI key is read from env via `app/app_secrets.py` (`openai_api_key`).
